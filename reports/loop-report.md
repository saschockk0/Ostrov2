## 2026-06-06 — итерация #2

**Цель цикла:** двинуть приоритет #1 «Флот → карточки-маркетплейс (листалка фото)».

**Что сделано**

- **Реализована карусель фото внутри карточки флота.** Раньше карточка показывала только первое фото, остальные открывались только в полноэкранном лайтбоксе. Теперь:
  - Все фото рендерятся в карточке стопкой, активное — `opacity:1`, остальные `opacity:0` с плавным переходом 0.25s.
  - Точки-индикаторы (`role="tablist"`, dots внизу медиа-области) появляются только если фото больше одного.
  - Клик по точке переключает активное фото и **не открывает** лайтбокс (`stopPropagation`).
  - Клик по самой карточке открывает лайтбокс **на текущем фото карусели** (а не на первом).
  - Стрелки ← / → на клавиатуре переключают фото в фокусированной карточке.
  - `aria-selected` обновляется на точках, у каждого `<img>` свой `alt` («Катамаран — фото 2»).
  - `loading="eager"` только на первом фото, остальные — `lazy`.

**Затронутые файлы**

- [public/app.js:853–874](public/app.js:853) — `renderFleetCard`: вместо одного `<img>` теперь все фото + dots.
- [public/app.js:935–966](public/app.js:935) — обработчики: dot.click → setActive, card.click → lightbox at activeIndex, keyboard arrows.
- [public/styles.css:471–516](public/styles.css:471) — `.va-fleet-card__img` (стэк с opacity transition), `.va-fleet-card__dots`, `.va-fleet-card__dot` + active/hover/focus.

**Проверка** (preview server :3000, мок-данные через `renderFleetSection`)

- 3 карточки: Hobie 16 (3 фото) → 3 точки, Лазер (2 фото) → 2 точки, Плот (1 фото) → 0 точек. ✓
- Клик по точке 3 в первой карточке: активный `<img>` сменился с index 0 на 2, точка 2 получила `is-active` + `aria-selected="true"`. ✓
- Клик по точке 2 → клик по карточке: лайтбокс открылся на `/content/regata.jpg` (фото index 1), а не на первом. ✓
- Single-photo карточка не получила точек — поведение клика как раньше. ✓
- Скриншот: dots компактные, polished, поверх медиа, с blur-фоном.

## 2026-06-06 — итерация #1

**Цель цикла:** запуск /loop, маленький, но видимый шаг по приоритету «Флот».

**Что сделано**

- **UX-улучшение секции «Флот».** Раньше пока шла загрузка `/api/fleet` — сетка была пустой; на ошибке сети показывалось то же «Информация о флоте скоро появится», что и при пустой БД. Пользователь не мог отличить «нет данных» от «не загрузилось», и не было способа повторить.
- Добавлены три состояния:
  1. **Skeleton-loader** — 3 карточки-плейсхолдера с shimmer-анимацией, отрисовываются сразу при старте `loadFleet()`.
  2. **Empty-state** — «Информация о флоте скоро появится» в аккуратной dashed-рамке.
  3. **Error-state** — «Не удалось загрузить флот. Проверьте соединение и попробуйте снова.» + кнопка `Повторить`, которая запускает `loadFleet()` заново.
- Скелетон-карточки игнорируются обработчиками клика по флоту (`:not(.va-fleet-card--skeleton)`), `aria-hidden="true"`.
- Уважается `prefers-reduced-motion` — shimmer выключается.

**Затронутые файлы**

- [public/app.js:844–921](public/app.js:844) — `renderFleetSkeleton`, `renderFleetError`, `loadFleet`, разделение empty vs error в `renderFleetSection`.
- [public/styles.css:609–668](public/styles.css:609) — `.va-skeleton`, `.va-fleet-card--skeleton`, `.va-fleet-empty`, `.va-fleet-empty__retry`, `@keyframes vaSkeletonShimmer`.

**Проверка**

- Запущен dev-server `ostrov-server` через preview, `GET /api/fleet` отвечает 500 (БД не сконфигурирована локально) — это позволило сразу проверить error-path.
- DOM-снимок: error-state с кнопкой `#fleetRetryBtn` отрисовался.
- Клик по `Повторить` → `loadFleet()` → 3 `.va-fleet-card--skeleton` в DOM → после fail обратно error-state. Цикл работает.
- Скриншоты: оба состояния визуально выглядят чисто, соответствуют дизайн-системе (синий outline-кнопка, dashed-граница ink-200).

**Не сделано / на будущие циклы**

- Сам carousel по фото внутри карточки (приоритет #1 «листалка фото + характеристики поверх») — отложено, требует более крупного рефакторинга `renderFleetCard` + новых стилей + touch-свайпов.
- Раздел «Наша команда» (приоритет #2) — нужен фактический контент от заказчика, нельзя выдумать.
- Отзывы Яндекса (`#reviewPhotos`) — нужно отдельное исследование OAuth-токена.

**Коммит/пуш:** не делалось (по инструкции /loop — без коммита).
