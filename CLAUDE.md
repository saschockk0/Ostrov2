# CLAUDE.md

## Проект

Одностраничный сайт Парусного Клуба «Остров» с ключевой целью: получить заявку через калькулятор и пошаговый опросник. Продакшн-сайт pkostrov.ru.

## Технический стек

| Слой | Технология |
|------|-----------|
| Frontend | HTML5 + CSS3 + Vanilla JS (без фреймворков, без бандлера) |
| Backend | Node.js + Express 5 |
| БД | MySQL / MariaDB (через `mysql2`) |
| Почта | Nodemailer (SMTP) |
| Платежи | YooKassa СБП |
| Антиспам | Cloudflare Turnstile + honeypot + rate-limit |
| Деплой | PM2 (`ecosystem.config.js`) |

> **Важно:** В CLAUDE.md изначально упоминался SQLite, но проект перешёл на **MySQL**. Обёртка в `database.js` имитирует SQLite API (`.run`, `.get`, `.all`), но работает поверх `mysql2`.

## Пошаговый опросник (Wizard)

7 шагов, реализованы в `public/app.js`. Файлы: `public/index.html` (разметка шагов), `public/styles.css` (стили модала).

| Шаг | Содержание | Обязательные поля |
|-----|-----------|------------------|
| 1 | Гости и даты | ≥1 гость, диапазон дат |
| 2 | Снаряжение | — |
| 3 | Активности (SUP, каяк) | — |
| 4 | Доп. услуги | — |
| 5 | Контакты | Имя, телефон |
| 6 | Сводка + CAPTCHA + согласие | Чекбокс конфиденциальности |
| 7 | Подтверждение + опция предоплаты | — |

**Особенности:**
- Конфигурация (гости, даты, не ПДн) сохраняется в `sessionStorage` (ключ `ostrov_trip_v1`)
- Калькулятор на лендинге предзаполняет шаг 1
- Живой бар стоимости (`quoteBar`) отображается на всех шагах со 2-го
- Серверный перерасчёт `POST /api/quote` выполняется перед отправкой заявки

## Антиспам

1. **Cloudflare Turnstile** — верифицируется на сервере перед сохранением заявки; при отсутствии `TURNSTILE_SECRET_KEY` пропускается.
2. **Honeypot** — скрытое поле `name="website"`; заявки с заполненным полем отклоняются (400).
3. **Rate limiting** — `express-rate-limit`:
   - Общий API: 120 req / 10 мин
   - Отправка заявки: 8 req / 10 мин / IP
   - Инициирование платежа: 10 req / 10 мин / IP
   - Вход в admin: 10 попыток / 15 мин
4. **Валидация** — длина полей, минимум 10 цифр в телефоне, обязательные поля проверяются на сервере.
5. **Проверка доступности** — перед сохранением заявки проверяется наличие свободных мест; при переполнении возвращается 409.

## Платежи (YooKassa СБП)

Документация: `PAYMENTS.md`.

**Сценарий 1 — Клиентская предоплата:**
1. Клиент нажимает «Внести предоплату» на шаге 7
2. `POST /api/applications/:id/payment` → YooKassa API → `confirmationUrl`
3. Редирект клиента на страницу СБП (QR-код или выбор банка)
4. Webhook `POST /api/webhooks/yookassa` обновляет статус платежа

**Сценарий 2 — Ручное выставление счёта:**
Менеджер в admin-панели формирует ссылку оплаты для отправки клиенту.

**Процент предоплаты:** `SBP_DEFAULT_PREPAY_PERCENT` (по умолчанию 30%).

## Ключевые принципы разработки

1. **Логика цен — только на backend.** Клиент показывает предварительный расчёт, сервер пересчитывает перед сохранением. Источник истины — `data/prices.json`.
2. **Без фреймворков.** Ни React, ни Vue, ни Tailwind — только ванильный HTML/CSS/JS. Не добавлять без явной договорённости.
3. **Без бандлера.** Нет webpack/vite/rollup. Файлы подключаются напрямую через `<script src>`.
4. **Минимум зависимостей.** Только необходимые npm-пакеты.
5. **Простота и читаемость** важнее абстракций.
6. **Не отправлять заявку** без обязательных полей и валидных дат.
7. **Адаптивность** — точки перелома 768px и 1024px.
8. **ПДн в sessionStorage не хранить** — только конфигурация поездки (гости, даты).
9. **Выходные дни** для расчётов: **пятница, суббота, воскресенье**.

## Особые замечания для AI-ассистентов

- **БД — MySQL, не SQLite.** Несмотря на первоначальное упоминание SQLite в документации.
- **Telegram ещё не реализован** — есть TODO в `src/server.js`.
- **Turnstile опционален** — без ключа CAPTCHA не проверяется, что корректно для dev-окружения.
- **Цены могут меняться** в `data/prices.json` без изменения кода — именно для этого они вынесены в файл.
- **Расчёт стоимости** в `src/pricing.js` является единственным авторитетным источником; клиентский расчёт в `app.js` — только для UX-предпросмотра.
- **Admin-маршруты** монтируются на `/ostrov-admin` (не `/admin`) для безопасности.
- **`variant-a.css`** — альтернативная стилизация для A/B-тестирования, не является основной.
